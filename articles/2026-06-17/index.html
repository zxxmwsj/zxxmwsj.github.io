Export Controls on AI Models Undermine US Cyber Defense ⭐️ 8.0/10

The US government’s export controls on AI models like Anthropic’s Claude Fable 5 are counterproductive because they prevent the model from fixing security vulnerabilities, as highlighted by cybersecurity expert Kate Moussouris. The controls were triggered by a supposed ‘jailbreak’ where the model was asked to fix code with known CVEs, which is actually a defensive use case. This situation reveals a fundamental misunderstanding by policymakers: the same capability that could be used for offensive cyber attacks is essential for defensive security. Restricting AI models’ ability to fix code weakens US cyber defense and sets a dangerous precedent for AI regulation. The ‘jailbreak’ involved researchers asking Fable 5 to ‘fix this code’ containing known CVEs and deliberately planted vulnerabilities, which the model did through a multistep manual process. Kate Moussouris confirmed that this is exactly what defenders need—finding, fixing, and testing patches—and that removing this capability would make the model worse at bug fixing.

rss · Simon Willison · Jun 16, 05:20

Background: Export controls on AI models aim to prevent adversaries from using advanced AI for malicious purposes, such as developing cyber weapons. However, the same models are also critical for defensive cybersecurity tasks like identifying and patching vulnerabilities. The case of Claude Fable 5 illustrates the tension between security through restriction and security through capability.

References
  • AI Model Outputs Demand the Attention of Export Control Agencies

Discussion: The discussion on Simon Willison’s blog highlights frustration with non-technical policymakers who conflate defensive and offensive AI uses. Commenters agree that the export control is misguided and that models should be encouraged to fix bugs, not penalized for it.

quicktok: A Faster BPE Tokenizer Byte-Identical to tiktoken ⭐️ 8.0/10

quicktok is a new C++ BPE tokenizer that achieves 2–11x speedup over tiktoken and 2–3.6x over bpe-openai while producing byte-identical token IDs. It ships with pretrained vocabularies for cl100k, o200k, GPT-OSS, Llama-3, and Qwen2.5/3. Tokenization is a bottleneck in many NLP pipelines, and quicktok’s speed improvements can reduce latency and cost for large-scale text processing. Its exact compatibility with tiktoken ensures seamless integration with existing OpenAI-based workflows. The speedup comes from data structure engineering: a 2-byte trie for longest-match walk, dense exactly-keyed caches for merge-validity checks, and a hand-compiled pretokenizer instead of a general regex engine. Benchmarks on Apple M1 show 121.7 MB/s for cl100k_base on The Pile dataset.

reddit · r/MachineLearning · /u/casa_nova · Jun 16, 04:24

Background: BPE (Byte Pair Encoding) is a tokenization algorithm that breaks text into subword units based on frequency, widely used in LLMs like GPT-4 and Llama. tiktoken is OpenAI’s official BPE tokenizer, but its Python implementation can be slow for high-throughput applications. quicktok is a drop-in replacement that maintains exact byte-level compatibility.

References
  • GitHub - openai/tiktoken: tiktoken is a fast BPE tokeniser for use with OpenAI's models. · GitHub
  • Byte-pair encoding - Wikipedia

Discussion: The Reddit discussion praised the project’s thorough benchmarks and technical depth, with commenters noting the clever use of a 2-byte trie and hand-compiled pretokenizer. Some asked about multi-threading support and integration with popular frameworks like Hugging Face tokenizers.

Tags: #tokenization, #BPE, #performance, #NLP, #open-source

Stop Using JWTs for Browser Sessions ⭐️ 7.0/10

A controversial post argues against using JSON Web Tokens (JWTs) for browser-based user sessions, citing security concerns and advocating for traditional session cookies instead. This debate affects how developers choose authentication methods for web applications, potentially shifting best practices away from JWTs for session management and toward server-side sessions. Critics note that JWTs often lack proper revocation mechanisms and can be vulnerable to storage-based attacks, while proponents argue that short-lived tokens with refresh flows mitigate these risks.

hackernews · dzonga · Jun 16, 16:49 · Discussion

Background: JSON Web Tokens (JWTs) are a stateless authentication method where user claims are encoded in a signed token. They are widely used for service-to-service communication but have been criticized for browser sessions due to difficulties in revocation and secure storage. Traditional session cookies rely on server-side state and are generally considered more secure for browser use.

Slay the Spire 2 Uses Custom PRNG for Cross-Platform Seeds ⭐️ 7.0/10

Slay the Spire 2 implements a custom pseudo-random number generator (PRNG) to ensure that seeds produce identical game outcomes across all platforms, avoiding the platform-specific RNG inconsistencies that plagued the original game. This change guarantees that players can share seeds and experience the same gameplay regardless of platform, and it prevents future updates from breaking seed reproducibility, which is critical for the game’s community and competitive scene. The custom PRNG replaces C#’s System.Random, which varied across platforms and could change with .NET updates. The article also discusses how correlated randomness affects game balance, such as the probability of receiving a specific curse from Neow’s Bones.

hackernews · rdmuser · Jun 16, 09:46 · Discussion

Background: Pseudo-random number generators (PRNGs) are algorithms that produce sequences of numbers that appear random but are deterministic given an initial seed. In games like Slay the Spire, seeds allow players to replay the same sequence of events. However, different platforms or language runtime versions may implement PRNGs differently, causing seeds to produce different outcomes. By implementing a custom PRNG, developers gain full control over the algorithm, ensuring consistency across all environments.

References
  • Correlated randomness in Slay the Spire 2 - Andy Tockman

Discussion: Commenters appreciated the technical deep dive and noted that the original Slay the Spire had an unwinnable seed due to RNG issues. Some pointed out that Godot’s GDScript uses PCG32, which would avoid this problem, but the game uses C# System.Random instead.

Bash /dev/tcp Enables HTTP Without curl ⭐️ 6.0/10

A blog post demonstrates that Bash’s built-in /dev/tcp feature can be used to make raw HTTP requests, providing a lightweight alternative to curl or wget in minimal environments. This trick is valuable for debugging in constrained environments like Docker containers or minimal Linux systems where curl and wget are unavailable, enabling basic HTTP connectivity checks without additional tools. The technique uses Bash’s /dev/tcp virtual filesystem to open a TCP connection, then sends a raw HTTP request via printf and reads the response with cat. It only supports HTTP/1.1 and does not handle HTTPS, redirects, or proper HTTP parsing.

hackernews · mrshu · Jun 16, 16:40 · Discussion

Background: Bash’s /dev/tcp is a compile-time option that enables TCP connections through file descriptors, acting as a virtual device. It is not enabled in all Bash builds (e.g., some systems disable it for security). This feature allows basic network interactions without external tools like netcat.
